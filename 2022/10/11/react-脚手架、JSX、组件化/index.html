I. 什么是react

用于构建用户界面的javascript库
起源于 facebook

特点

1. 声明式： 只要描述结构，渲染ui过程由react来做的
2. 基于组件： 最重要
3. 学习一次，随处使用： 开发web应用、 开发移动端原生应用、 还有vr开发

II. react 基本使用

1. createElement
  • 1. const title = React.createElement('p', {title: '属性'}, id: 'p1' )

    第一个参数：要创建的react元素名称

    第二个参数： 元素的属性（可以多个）

    第三个参数及以后： 该元素的子节点

2. ReactDOM.render
  • ReactDOM.render (el, document.getElementById('root'))

    第一个参数：要创建的react元素名称

    第二个参数： DOM对象， 用于渲染到页面中的位置

III. react 脚手架的使用

利用脚手架初始化项目

  • 1. 初始化项目： npx create-react-app my-app
  • 2. 启动项目： npm start

npx命令介绍

  • 1. 原先： 先全局安装脚手架（全局安装长时间不用没有意义）， 才能使用脚手架提供的命令初始化项目
  • 2. 现在： 无需全局安装脚手架， 可直接使用脚手架提供的命令
  • 3. 推荐： npx create-react-app my-app

yarn命令介绍

  • 1. 起源： facebook发布
  • 2. 特点： 快速、安全、可靠
  • 3. 安装： yarn add 包名称

导入react

  • 1. 导入 react 和 react-dom包
  • import React from 'react' import ReactDOM from 'react-dom'
  • 2. 调用 React.createElement创建元素
  • 3. 调用 ReactDOM.render 渲染元素到页面中

IV. JSX学习

createElement()问题

  • 1. 繁琐不简洁
  • 2. 不直观， 一眼看不出结构
  • 3. 不优雅， 用户体验不爽

jsx学习

  • 1. 是javascript xml简写
    2. 优势： 声明式语法更直观
    3. 优势： 降低了学习成本， 提高了开发效率
    4. jsx 是react 的核心内容
    5. jsx 表示在js代码中写html, 是react 声明式的体现
    6. react 完全利用js语言自身的能力来编写UI， 而不是造轮子的方式增强html功能
  • 1. jsx创建元素： const title = 《h1》 新建元素 《/h1》
  • 2. 渲染元素： ReactDOM.render(title, document.getElementById('root'))

为什么脚手架可以使用jsx语法

  • 1. jsx 不是ecmascript 标准语法， 是ecmascript 的语法扩展
  • 2. 需要babel 编译处理，才能在浏览器环境中使用
  • 3. create-react-app 中已经默认有该配置， 所以无需手动配置
  • 4. 编译包： @babel/preset-react

jsx语法注意事项

  • 1. 属性名： 驼峰命名法
  • 2. 特殊属性： class -> className; for -> htmlFor; tabindex ->tabIndex
  • 3. 推荐： 小括号包裹 ‘ jsx ’，避免自动插入分号

jsx 嵌入js 表达式

  • 定义： { js表达式 }

jsx 条件渲染 、列表渲染 、 样式处理

  • 1. 条件渲染： if else / 三元表达式 / 逻辑与运算符
  • 2. 列表渲染： 数组的 map 方法 （避免使用索引号作为key）
  • 3. 样式处理：

    ● 行内样式 - style

    style = 双括号， 属性之间逗号隔开

    ● 类名 - className

    className="bg"

V. 组件

组件化-组件特点

  • 1. 特点： 可复用、 独立 、 可组合

组件化-组件的两种创建方式

  • 1. 函数创建组件: 用函数（或肩头函数）创建的组件
    a、 约定1： 函数名称 - 必须大写字母开头
    b、 约定2： 必须有 返回值
    c、 约定3： 返回值为 null, 不渲染任何内容
    d、 渲染组件： ReactDOM.render(<组件名>, document.getElementById('root'))
  • 2. class创建组件: 用ES6的class创建的组件
    a、 约定1： 类名称 - 必须大写字母开头
    b、 约定2： 类组件 应该继承React.Component父类， 从而可以使用父类中提供的方法和属性
    c、 约定3： 必须有 render（）方法
    d、 约定4： render方法必须有 返回值
    e、 渲染组件： ReactDOM.render(<组件名>, document.getElementById('root'))
  • 3. 组件抽离： 放到独立的js文件中
    a、 第一步： 创建 js 文件
    b、 第二步： js文件中 - 导入React
    c、 第三步： js文件中 - 创建组件
    d、 第四步： js文件中 - 导出该组件
    e、 第五步： 需要使用的js中 - 导入该组件
    f、 第六步： 需要使用的js中 - 渲染该组件

组件化-组件的事件处理

  • 1. 事件绑定: 与DOM事件语法相似
    a、 语法： on + 事件名称 = { 事件处理程序 }
    b、 注意： 事件采用驼峰命名法
    c、 注意： ` 函数组件 `的事件处理程序 - 没有this ( onClick = {handelClick} )
  • 2. 获取事件对象： e
    a、 渠道： { 事件处理程序（e） } 的 参数
    b、 名称： react中的事件对象叫 - 合成事件（对象）
    c、 特点： ` 合成对象 `的事件处理程序 - 兼容所有浏览器
    d、 事件处理程序： 这里面的` this ` 是undefined

组件化-有状态组件 和 无状态组件

状态： state -> 数据

  • 1. 无状态组件: 函数组件
    a、 定义： 没有自己的状态， 只负责数据的展示 (需要配合hooks才能更新UI)
  • 2. 有状态组件： 类组件
    a、 定义： 有自己的状态 负责更新UI

组件化-state和 setState

  • 1. state基本使用
    a、 定义： state即数据 组件内部私有数据
    b、 结构： state值是对象
    c、 获取状态： this.state.value
  • 2. setState()修改状态 : 思想 - 数据驱动视图的体现
    a、 定义： 状态是可变的
    b、 语法： this.setState({ 修改的key: 修改后的value})
    c、 注意： 不能直接修改state中的值
    d、 作用： 修改state 和 更新 ui

组件化-事件绑定中 this的指向解决方案

  • 1. 箭头函数
    a、 特点： 箭头函数有自身不绑定 this的特点， 取决于外部环境
    b、 外部环境： 这里指向了 render()方法， render中的this为组件实例，可以获取setState()

    ... onClick = { () => { this.handelClick }} ...

  • 2. bind方法 : Function.prototpe.bind
    a、 定义： 利用ES5中的bind方法， 将事件处理程序中的 “this” 与组件实例绑定在一起
    b、 语法： this.handelClick = this.handelClick.bind(this)

    class Hello extends React.Component {

    constructor() {

    super()

    this.handelClick = this.handelClick.bind(this)

    }

    render() {

    return(

    ... onClick = { this.handelClick } ...

    )

    }

    }

  • 3. class实例方法 : 箭头函数形式的class 实例方法
    a、 定义： 利用箭头函数形式的class实例方法
    b、 注意： 语法是试验性的语法， 但是由于有babel的存在可以直接使用

    class Hello extends React.Component {

    constructor() {

    super()

    }

    handelClick = () => {

    console.log('111')

    }

    render() {

    return(

    ... onClick = { this.handelClick } ...

    )

    }

    }
