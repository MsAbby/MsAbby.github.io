I. 组件通信

props

  • 1. props作用： “ 接收 ”传递给组件的数据
  • 2. props特点：
    1、 可以传递任意类型的数据
    2、 只读对象
    3、 针对类组件： 若写构造函数，要将props传给super(props), 否则无法在构造函数constructor中获取props
  • 3. 传递数据：给组件标签添加属性
  • 4. 接收数据：
    1、 函数组件 - 通过函数参数props接收
    2、 类组件 - 通过this.props接收数据
  • 5. props深入：
    1、 children属性：
    ● 表示组件标签的子节点。 标签有子节点，props就具有该属性
    ● 值-可以使任意值（文本，组件、函数、jsx等）
    2、 props默认值：defaultProps
    ● 写法： 组件名.defaultProps = { pageSize: 10 }
    ● 作用： 在未传props时生效
    3、 props校验： propTypes
    ● 方式： 创建组件时， 就指定props 的类型和格式
    ● 作用： 捕获错误、 增加组件的健壮性
    ● 作约束规则： array、bool、func、number、object、string
    ● 必填项： isRequiRed
    ● 特定结构的对象： shape({})
    ● 第一步： 安装包: prop-types
    ● 第二步： 导入prop-types包（import propTypes from 'prop-types'）
    ● 第三步： 添加校验规则： 组件名.propTypes = { color: propTypes.array.isRequiRed}

组件通信三种基本方式

  • 1. 父组件 - 子组件：
    1、 准备数据： 父组件中提供数据state
    2、 传递方式： 子组件的属性传递该数据
    3、 获取数据： 子组件通过 props 获取
  • 2. 子组件 - 父组件：
    1、 思路： 利用回调函数
    2、 准备回调函数： 父组件提供回调函数（用于接收数据）
    3、 传递方式： 子组件的属性传递
    4、 子组件传参： 子组件利用this.props.属性名（传递的数据）
  • 3. 兄弟组件：
    1、 状态提升思路： 将” 共享状态 “ 提升到 最近的“ 公共父组件 ” 管理该状态
    2、 提供共享状态： 公共父组件提供 「 共享状态 」
    3、 提供共享状态方法 公共父组件提供 「 可操作共享状态的方法 」/div>
    4、 接收数据： 子组件通过props 「 接收状态 」和 「 接收 操作状态的方法 」

  • 1. 作用： 跨组件传递数据（远方亲戚-嵌套多层）
  • 2. 使用步骤：

II. 组件的生命周期

组件的生命周期 - 概述

  • 1. 组件生命周期： 组件被创建 到 挂载到页面运行， 再到组件不用时卸载的过程
  • 2. 钩子函数： 生命周期每个阶段会有一些方法被调用，这些方法是「钩子函数」
  • 3. 钩子函数作用： 为开发人员在不同阶段的操作提供了时机
  • 4. 只有类组件有生命周期

组件的生命周期 - 三个阶段

  • 1. 创建时（挂载阶段）：
    ❉ 执行时机： 组件创建时
    ❉ 执行顺序： constructor() - render() - componentDidMount()
    钩子函数 执行时机 作用
    constructor 最先执行、组件创建时 初始化state、 为事件程序绑定this
    render 每次组件渲染都会触发 渲染UI(不能调用setState)
    componentDidMount 组件挂载后（完成dom渲染） 发送网络请求、 进行dom操作
  • 2. 更新时：
    11
  • 3. 卸载时
    11

为什么脚手架可以使用jsx语法

  • 1. jsx 不是ecmascript 标准语法， 是ecmascript 的语法扩展
  • 2. 需要babel 编译处理，才能在浏览器环境中使用
  • 3. create-react-app 中已经默认有该配置， 所以无需手动配置
  • 4. 编译包： @babel/preset-react

jsx语法注意事项

  • 1. 属性名： 驼峰命名法
  • 2. 特殊属性： class -> className; for -> htmlFor; tabindex ->tabIndex
  • 3. 推荐： 小括号包裹 ‘ jsx ’，避免自动插入分号

jsx 嵌入js 表达式

  • 定义： { js表达式 }

jsx 条件渲染 、列表渲染 、 样式处理

  • 1. 条件渲染： if else / 三元表达式 / 逻辑与运算符
  • 2. 列表渲染： 数组的 map 方法 （避免使用索引号作为key）
  • 3. 样式处理：

    ● 行内样式 - style

    style = 双括号， 属性之间逗号隔开

    ● 类名 - className

    className="bg"

V. 组件

组件化-组件特点

  • 1. 特点： 可复用、 独立 、 可组合

组件化-组件的两种创建方式

  • 1. 函数创建组件: 用函数（或肩头函数）创建的组件
    a、 约定1： 函数名称 - 必须大写字母开头
    b、 约定2： 必须有 返回值
    c、 约定3： 返回值为 null, 不渲染任何内容
    d、 渲染组件： ReactDOM.render(<组件名>, document.getElementById('root'))
  • 2. class创建组件: 用ES6的class创建的组件
    a、 约定1： 类名称 - 必须大写字母开头
    b、 约定2： 类组件 应该继承React.Component父类， 从而可以使用父类中提供的方法和属性
    c、 约定3： 必须有 render（）方法
    d、 约定4： render方法必须有 返回值
    e、 渲染组件： ReactDOM.render(<组件名>, document.getElementById('root'))
  • 3. 组件抽离： 放到独立的js文件中
    a、 第一步： 创建 js 文件
    b、 第二步： js文件中 - 导入React
    c、 第三步： js文件中 - 创建组件
    d、 第四步： js文件中 - 导出该组件
    e、 第五步： 需要使用的js中 - 导入该组件
    f、 第六步： 需要使用的js中 - 渲染该组件

组件化-组件的事件处理

  • 1. 事件绑定: 与DOM事件语法相似
    a、 语法： on + 事件名称 = { 事件处理程序 }
    b、 注意： 事件采用驼峰命名法
    c、 注意： ` 函数组件 `的事件处理程序 - 没有this ( onClick = {handelClick} )
  • 2. 获取事件对象： e
    a、 渠道： { 事件处理程序（e） } 的 参数
    b、 名称： react中的事件对象叫 - 合成事件（对象）
    c、 特点： ` 合成对象 `的事件处理程序 - 兼容所有浏览器
    d、 事件处理程序： 这里面的` this ` 是undefined

组件化-有状态组件 和 无状态组件

状态： state -> 数据

  • 1. 无状态组件: 函数组件
    a、 定义： 没有自己的状态， 只负责数据的展示 (需要配合hooks才能更新UI)
  • 2. 有状态组件： 类组件
    a、 定义： 有自己的状态 负责更新UI

组件化-state和 setState

  • 1. state基本使用
    a、 定义： state即数据 组件内部私有数据
    b、 结构： state值是对象
    c、 获取状态： this.state.value
  • 2. setState()修改状态 : 思想 - 数据驱动视图的体现
    a、 定义： 状态是可变的
    b、 语法： this.setState({ 修改的key: 修改后的value})
    c、 注意： 不能直接修改state中的值
    d、 作用： 修改state 和 更新 ui

组件化-事件绑定中 this的指向解决方案

  • 1. 箭头函数
    a、 特点： 箭头函数有自身不绑定 this的特点， 取决于外部环境
    b、 外部环境： 这里指向了 render()方法， render中的this为组件实例，可以获取setState()

    ... onClick = { () => { this.handelClick }} ...

  • 2. bind方法 : Function.prototpe.bind
    a、 定义： 利用ES5中的bind方法， 将事件处理程序中的 “this” 与组件实例绑定在一起
    b、 语法： this.handelClick = this.handelClick.bind(this)

    class Hello extends React.Component {

    constructor() {

    super()

    this.handelClick = this.handelClick.bind(this)

    }

    render() {

    return(

    ... onClick = { this.handelClick } ...

    )

    }

    }

  • 3. class实例方法 : 箭头函数形式的class 实例方法
    a、 定义： 利用箭头函数形式的class实例方法
    b、 注意： 语法是试验性的语法， 但是由于有babel的存在可以直接使用

    class Hello extends React.Component {

    constructor() {

    super()

    }

    handelClick = () => {

    console.log('111')

    }

    render() {

    return(

    ... onClick = { this.handelClick } ...

    )

    }

    }

组件化-表单处理

  • 1. 受控组件: 其值 受到react的控制
    a、 关于传统表单： html 中表单元素可以输入的， 即可以维护自己的可变状态
    b、 关于react数据： react 可变数据同意保存在 state中， 通过setState 维护可变状态
    c、 结合： 将react 的 state 和 表单元素的 value 绑定在一起由state的值来控制表单元素的值

    ... type="text" value={ this.state.value } onChange={ this.handelChange}...

    d、 有多个表单元素时， 优化表单元素的步骤, 一个事件处理程序处理多个表单元素
    1、 第一步： 给表单元素添加 name属性， 名称与state相同
    2、 第二步： 根据 元素类型 获取对应属性及其值
    3、 第三步： change事件处理程序中通过 [name] 来修改对应的state

    handelChange = e => {

    const target = e.target

    // 根据类型获取值

    const value = target.type === 'checkbox' ? target.checked: target.value

    // 获取name

    const name = target.name

    this.setState({

    [name]: value

    })

    }

  • 2. 非受控组件： 借助ref 使用原生DOM方式来获取表单元素值
    a、 ref： 获取 DOM 对象 或组件
    b、 使用步骤：
    1、 第一步： 创建ref 对象 - React.createRef（）

    constructor(){

    super()

    this.txtRef = React.createRef()

    }

    2、 第二步： 将创建好的ref对象 关联到表单元素上

    ... type="text" ref={this.txtRef}...

    3、 第三步： 通过ref 对象获取到文本狂的值

    this.txtRef.current.value
